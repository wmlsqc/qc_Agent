# 规划模块说明

`planning/` 目录提供当前项目的轻量规划能力。它只负责识别用户问题类型，并给出推荐工具顺序和回答重点，不执行工具，也不替代 ReactAgent。

## 主要文件

- `simple_planner.py`：基于关键词的轻量任务分类和规划提示。

## 当前职责

- 判断当前用户问题属于哪类任务。
- 给出建议调用的工具顺序。
- 给出最终回答需要关注的重点。
- 在工具失败或没有数据时，提供兜底策略。

## 当前支持的任务类型

| 任务类型 | 典型问题 | 推荐工具方向 |
| --- | --- | --- |
| 综合分析类 | 整体使用情况、本月表现、有没有需要注意的问题 | 用户 ID、用户画像、设备、耗材、清扫历史、RAG |
| 故障诊断类 | 漏扫、不充电、异响、不出水、无法回充 | 用户 ID、故障诊断、RAG |
| 设备状态类 | 电量、在线状态、当前模式、异常状态 | 用户 ID、设备状态 |
| 耗材类 | 主刷、边刷、滤网、拖布、尘袋、更换 | 用户 ID、耗材状态、用户画像 |
| 清扫历史类 | 清扫记录、清扫效果、覆盖率、漏扫次数 | 用户 ID、清扫历史、用户画像 |
| 知识问答类 | 怎么保养、如何选择、为什么会这样 | RAG 知识库 |

## 使用位置

`context/context_builder.py` 会调用本模块生成轻量规划上下文，然后由 `services/chat_service.py` 注入给 ReactAgent 作为参考。

## 当前限制

该模块是基于关键词的轻量规划，不是复杂的多智能体规划系统。它适合演示和规则清晰的业务场景，后续可以升级为更细粒度的意图识别、工具路由或可观测的规划链路。
